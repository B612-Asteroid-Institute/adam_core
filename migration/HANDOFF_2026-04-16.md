# ADAM Core Rust Migration Handoff (2026-04-16)

## Mission and Scope
- Primary objective: migrate `adam-core` to an API-majority Rust-backed implementation via PyO3 + maturin, while preserving Python surface continuity.
- Migration unit policy: prioritize high-level atomic public entrypoints; do not port every helper unless it is externally used or required for measurable gains.
- Current priority: coordinate transformation paths.
- High-level execution rule: once a supported high-level API crosses Python -> Rust, keep internal execution in Rust until final return (no Python<->Rust ping-pong).

## Canonical Sources (Read First)
- `migration/PLAN.md` (authoritative contract)
- `migration/TODO.md` (tracker)
- `src/adam_core/_rust/status.py` (single registry: per-API migration state, default backend, rust module, perf gate)
- `migration/waivers.yaml` (explicit retained-legacy governance)
- `decisions.md` (user decisions)
- `journal.md` (execution record, benchmark/test outcomes)

## Non-Negotiable Gates
- Rust default promotion requires:
  - parity tests passing
  - `>= 1.2x` speedup at both p50 and p95
- Full-suite requirement: run existing pytest suite with Rust backend required:
  - `ADAM_CORE_REQUIRE_RUST_BACKEND=1 pdm run pytest --benchmark-skip -m 'not profile'`
