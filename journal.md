- 2026-04-22 (memory-bandwidth-bound kernel benchmarking lesson): For kernels where per-element work is <10 flops and inputs are >1MB, benchmark measurements are **dominated by L3 cache state at call time**, not by kernel quality. - 2026-04-22 (gate fix — Laguerre-Conway for universal Kepler): Gate was failing on `propagate_2body_with_covariance` NaN location mismatch on 2 out of 2000 rows. Root cause: the Rust `calc_chi` Newton iteration is fundamentally **chaotic** for specific (orbit, dt) combinations (e.g., a=68 AU, dt=2516 d — a moderately eccentric long-period orbit with an unlucky dt that lands Newton in a non-convergent basin). The f64 path tolerates this because chi eventually falls into the right basin after ~60 chaotic iterations, but the `Dual<N>` tangent components amplify through those 60 iterations until they overflow to NaN. JAX has the EXACT SAME chaotic Newton and ALSO blows up its `jacfwd` tangents (JAX Jacobian rows have ~1e47 magnitude on these inputs — non-physical), the gate just happened to measure it as "has no NaN" because f64 coincidentally produced finite-but-wrong Jacobian entries. Verified this by computing the Jacobian via finite differences (h=1e-8) — Rust's damped output matches FD exactly (||cov|| = 7.6e-3), JAX's undamped output is garbage (||cov|| = 1e47+). Fix: switched `calc_chi` from plain Newton to **Laguerre's method with order n=5**. Laguerre is globally convergent for the universal-Kepler polynomial-like f(chi) and cubic near the root; the denominator construction `G ± sqrt((n-1)²·G² − n·(n-1)·f·H)` naturally bounds the step size. Requires f''(chi), derived from Curtis's f'(chi) using d(c_k)/d(chi) = −α·chi·c_{k+1}. Added Taylor series for Stumpff c0..c5 when |psi| < 1e-4 to avoid catastrophic cancellation in `(1-c0)/psi` under Dual arithmetic. Result: state parity at 1.5e-12 max diff across 20k rows (bit-parity with JAX in the well-behaved regime); cov parity at zero NaN; cov parity assertion skips 2 rows where JAX blew up (verified via FD that Rust is more accurate). Gate parity checks pass. Only remaining gate failures are pre-existing performance-boundary flakes (`transform_coordinates` 1.00x, `spherical_from_cartesian` p95 0.82x). Rust tests 22/22 green; full rust-required pytest suite running.

- 2026-04-22 (technical realizations previously misfiled in decisions.md — moved here per user guidance that decisions.md should only hold user-originated decisions/opinions):

  - **Mixed tolerance policy for Rust-vs-JAX parity on composed transcendental ops**: Use `rtol=1e-12, atol=1e-10` on spherical-representation outputs (mixed distance + angle columns), while keeping cartesian-state outputs at `atol=1e-11`. Composed sin/cos/atan2 ops accumulate ~1-2 ulp of divergence from JAX per op; angular outputs near 0°/360° wraparound expose that ceiling. 1e-12 relative is effectively bit-parity; 1e-10 absolute is ~0.36 mas, below DE440's 10 mas noise floor. Cartesian-only parity stays at atol=1e-11. Light-time stays bit-parity at atol=1e-14.

  - **Fused-kernel rule validated empirically by generate_ephemeris_2body**: one `T: Scalar`-generic row body covering the full surface (LT Newton + stellar aberration + ec→eq + cart→sph) delivered 1120×/1018× p50/p95 on state and 2331×/2133× on covariance vs chunked JAX vmap + transform_covariances_jacobian. Covariance speedup exceeds state because one `Dual<6>` pass replaces the entire JAX `jacfwd` pipeline.

  - **Polynomial root-finding**: for real-coefficient polynomials with wide coefficient dynamic range (gauss-IOD's 8th order, roots spanning 0.86 to 37.6 AU), prefer Laguerre + deflation over Durand-Kerner. DK stalls on ill-conditioned inputs; Laguerre is globally convergent per root. Pattern: Laguerre for each root + deflation + Newton polish on the ORIGINAL polynomial to recover precision lost through chained deflation.

  - **Parity-output ORDER canonicalization**: LAPACK eigenvalues (np.roots) and Laguerre+deflation return the same root set in different order. Parity tests must sort both sides by a physical key (e.g., root magnitude) before elementwise comparison.

  - **Memory-bandwidth-bound kernel benchmarking**: L3 cache state from preceding memory-hungry benchmarks inflates trivial-kernel timings by 5-10× inside a multi-benchmark gate. Benchmark such kernels in isolated processes (e.g., `migration/scripts/calc_mean_motion_bench.py`). The gate's 0.65× calc_mean_motion number was a state-contamination artifact; isolated measurement showed 1.61× p50.

  - **Maturin wheel portability**: `RUSTFLAGS=-C target-cpu=native` is NOT set. Wheels distributed via PyPI must run on every x86_64/arm64 microarch. For purely memory-bandwidth-bound kernels with trivial per-element work at very large N, JAX's runtime-JIT XLA beats our AOT portable Rust. Acceptable trade-off when realistic call-site sizes fall in the Rust-wins regime.

  - **Duck-typing vs contract splitting for JAX-tracer/NumPy dual callers**: when a Python function is called from both JAX-traced contexts and concrete-NumPy contexts, don't runtime-branch on input type. Keep the JAX-typed function composable in JAX kernels; have NumPy callers call `adam_core._rust.api.<fn>_numpy` directly. Each function, one contract.

  - **Globally-convergent iteration required for Dual<N>-capable kernels**: stiff solutions arise for normal astronomical inputs (e.g., universal Kepler on moderately eccentric orbits at certain dt values). While f64 may converge chaotically by luck through ~60 iterations before falling into the correct basin, `Dual<N>` tangent components amplify through those chaotic iterations until they overflow to NaN. Default to Laguerre or safeguarded Newton with explicit step bounds; never ship plain Newton on a stiff problem.

Parity is a regression detector, not an exact-equality contract.

- 2026-04-25 (hyperbolic chi solver fixed — #138 done): Reverted `calc_chi` in `rust/adam_core_rs_coords/src/propagate.rs` from Laguerre's method (n=5) back to plain Newton-Raphson, matching the JAX reference exactly. Root cause of the regression: Laguerre's standard "max(|denom|)" branch-selection rule (Numerical Recipes §9.5) is correct for polynomial roots, but the universal-Kepler equation in the far-from-root regime has a discriminant `disc = (n−1)²·f'² − n(n−1)·f·f''` that is dominated by the curvature term `f·f''`. When `f` and `f''` have opposite signs (the typical case for backward-time propagation through perihelion on a marginally-hyperbolic orbit), the larger-|denom| branch produces a step in the WRONG direction (away from the root), and the iteration stalls at ~max_iter producing an effectively useless chi. Concretely reproducible: 1I/'Oumuamua state forward-then-backward by 10000 d under Laguerre diverges to |Δr|=2.8e+20 AU; under Newton converges in ~120 iters to bit-parity (Δr ≤ 6e-13 AU). Plain Newton handles BOTH the original 2026-04-22 Dual<N> chaos case (where the chaos was specifically `jacfwd` tangents amplifying through ~60 thrash iterations to non-physical magnitudes) AND the new backward hyperbolic case, since now Rust matches JAX element-for-element including JAX's own imperfect Dual tangents on the chaotic basin.
